Week 0: Getting the lay of the land

Goal

This class will help you learn about the Structured Query Language, aka SQL. To do that, you will be immersed into the world of databases and the clients that talk to them, which can seem overwhelming for the first time. This lesson aims to ease you into that environment in bite-sized chunks. At the end of this lesson, you will have hands-on experience with many of the different tools we will use in this class.

Objectives

Check out the Konagora website; Konagora is a web resource developed by the author and supplements the book. I will be referring to it frequently.
Understand the roadmap of the book, SQL Success by Stéphane Faroult
Download and install XAMPP
Download and install SQL Workbench
Read the first two chapters of SQL Success

Details

Konagora

Konagora has a number of resources for both students and instructors. For those that just want to jump in, the SQL Sandbox is particularly useful - make sure you "Show/Hide" the tables as well. There are several supplemental features surrounding the book; for example, there are code samples from each chapter, sample databases for all popular types of databases, and a section for Errata (kinda small at the moment). There is a whole section for instructors which you will not see, including a section for taking a SQL test and doing sample homework problems which can be made available on a class-by-class basis.

Roadmap of "SQL Success"

Open up the Course Schedule as you do this. Look at the "Contents" of the book, read or skim the Preface. The first chapter gives a ton of historical context which I feel is outside the scope of this class, but could be useful for those who are completely lost about databases. (By the way, it is perfectly ok to be completely lost - this is an Intro class.) The class starts with Chapter 2; we will cover most but not all of the chapters, and even then, some chapters will only get partial coverage; because the author packs so much into each chapter, our normal pace will aim for half a chapter a week. It is important to note that Chapter 11 provides two examples of writing a very simple web-application, while Chapter 12 goes into more depth and provides more examples. It is my desire to model our Final Project on Chapter 12. We will see how close we get.

XAMPP

XAMPP stands for "Cross-platform Apache + MySQL + PHP + Perl". For this class, you will want to start Apache once, but 99% of your time will be with MySQL, and more specifically, writing queries. You don't have to worry too much about actually managing a database since that is covered in a different class (CSC 192). But inevitably, some of you will explore a little more and figure out how to use MySQL on your own. Likewise, some of you will be more interested in programming and will want to explore how to use PHP and/or perl to write an application to talk to their database.

SQL Workbench

SQL Workbench is a java-based container client used to talk to various kinds of databases. As a container client, it does not directly have the ability to talk to a database, but rather depends on the java libraries provided by the vendor of each database system. As such, if you want to connect to a mysql database, you will need a mysql jdbc connector. If you want Oracle, postgres, DB2, etc, you need the appropirate driver libraries for each. The purpose of the tool is to help you manage sql queries and provides limited functionality to explore the database. Note that there is another product called 'MySQL Workbench', which sounds very similar. It is a very different product meant only for MySQL, but essentially does exaclty the same thing.
