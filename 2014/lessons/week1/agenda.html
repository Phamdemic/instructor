Week 1: Connecting to a database, creating a table

Goal

At the end of this lesson, you have hands on experience with connecting to a database, creating a table and populating it with data. Even though students are not yet exposed to SQL proper, we are setting the stage for the rest of the semester.

Objectives

Connect to the class mysql database
Create a simple table
insert data into table

Details

Throughout this semester, I will refer to the "class database" or the "mysql on onyx". Onyx is the linux host where all of our class databases will live. Note that you may see "MariaDB" pop up on some of the screens and websites we will visit. For the curious, google MariaDB to learn more. For the sake of simplicity and continuity, I will refer to the database as mysql. And I wil not typically bother with the capital letters (MySQL). In class, you will connect to the database in two ways: 1) via the SQL Workbench installed on the class computer, and 2) via phpMyAdmin (hosted on onyx).

Connecting via SQL Workbench

Create a table

The author has provided the code from the book on his Konagora website. I have copied the mysql code, which you can use to copy'n'paste if you don't want to type it all out. Let's start with a basic table first: 

create table countries(country_code  char(2) not null primary key,
 country_name  varchar(50) not null,
 continent     varchar(20),
 unique(country_name));

Inserting data

And a relatively straight forward insert statement: 

insert into countries(country_code, country_name, continent)
values('us', 'United States', 'AMERICA'),
  ('in', 'India', 'ASIA'),
  ('gb', 'United Kingdom', 'EUROPE'),
  ('it', 'Italy', 'EUROPE');
