Week 4

Goal

Explore the differences between a single "flat" table (aka, denormalized) versus many smaller, related tables (aka, normalized).

Objectives

A. Run sql scripts to create a few extra tables that copy the data into a hierarchy.
B. Create a table from Question 1.3 of last week.
C. Convert all your queries from week 3 to work against these new tables, including the two Advanced questions.
D. Summarize your experience.

B. Create a new table for school names
"Translate the school names from their two-character abbreviation to the full name". Use a sql statement similar to: 

create table u4_school_names as ( <your query goes here> );
Name the first column school_id, the second column school_name (the second one goes after "end" of the CASE statement).

C. Rewrite Week 3 queries using new tables
You will probably want to explore the new tables a little. Do a few "select *" statements to see what they look like, how many rows they have, etc. Also, I have included column comments to better explain what each column does. If you are really observant, you will notice under the "Structure" tab in phpMyAdmin that the Primary Key icon is not gold for some columns.

Once you are familiar with the tables, rewrite the queries you wrote for section 1 last week using the new tables, including the two Advanced queries. Your results should be the same.

D. Summary
What differences did you observe between week3 and this week? Which one did you prefer to work with? Be sure to include any other thoughts/observations.
